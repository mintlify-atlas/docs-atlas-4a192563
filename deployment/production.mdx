## Overview

This guide covers deploying Enigma Developers to production using Docker Compose with security hardening, performance optimizations, and operational best practices.

## Production vs Development

The key differences between `docker-compose.yml` (development) and `docker-compose.prod.yml` (production):

### Port Binding

**Development** exposes ports on all interfaces:
```yaml
ports:
  - "4051:80"  # Accessible from any IP
  - "4050:4000"
```

**Production** binds ports to localhost only:
```yaml
ports:
  - "127.0.0.1:4051:80"  # Localhost only
  - "127.0.0.1:4050:4000"
```

<Warning>
  In production, services should only be accessible through a reverse proxy (Nginx), not directly exposed to the internet.
</Warning>

### Volume Configuration

**Development** uses local named volumes:
```yaml
volumes:
  pgdata:
```

**Production** uses external volumes for data persistence:
```yaml
volumes:
  pgdata:
    external: true
    name: app_pgdata
```

<Note>
  External volumes must be created before starting services in production.
</Note>

## Production Docker Compose

The complete `docker-compose.prod.yml` configuration:

```yaml docker-compose.prod.yml
services:
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: enigmasac-frontend
    restart: unless-stopped
    ports:
      - "127.0.0.1:4051:80"
    networks:
      - enigmasac-net

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: enigmasac-backend
    restart: unless-stopped
    env_file: .env
    ports:
      - "127.0.0.1:4050:4000"
    depends_on:
      db:
        condition: service_healthy
    networks:
      - enigmasac-net

  db:
    image: postgres:16-alpine
    container_name: enigmasac-db
    restart: unless-stopped
    env_file: .env
    environment:
      - POSTGRES_DB=${POSTGRES_DB}
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      interval: 5s
      timeout: 5s
      retries: 5
    networks:
      - enigmasac-net

volumes:
  pgdata:
    external: true
    name: app_pgdata

networks:
  enigmasac-net:
    driver: bridge
```

## Production Deployment Steps

<Steps>
  <Step title="Prepare the server">
    Ensure your server has the required software:
    
    ```bash
    # Install Docker
    curl -fsSL https://get.docker.com -o get-docker.sh
    sh get-docker.sh
    
    # Install Docker Compose
    sudo apt-get update
    sudo apt-get install docker-compose-plugin
    
    # Verify installations
    docker --version
    docker compose version
    ```
  </Step>

  <Step title="Clone the repository">
    Deploy your application code:
    
    ```bash
    git clone https://github.com/your-org/enigma-developers.git
    cd enigma-developers
    ```
  </Step>

  <Step title="Configure environment variables">
    Create and configure the production `.env` file:
    
    ```bash
    cp .env.example .env
    nano .env
    ```
    
    Update these critical values:
    
    ```bash
    NODE_ENV=production
    POSTGRES_PASSWORD=[generate strong 32+ char password]
    DATABASE_URL=postgresql://[user]:[password]@db:5432/enigma
    VITE_API_URL=https://enigmasac.com/api
    CORS_ORIGIN=https://enigmasac.com,https://www.enigmasac.com
    SMTP_PASS=[your SMTP app password]
    ```
    
    Secure the file:
    
    ```bash
    chmod 600 .env
    ```
  </Step>

  <Step title="Create external volume">
    Create the external PostgreSQL data volume:
    
    ```bash
    docker volume create app_pgdata
    ```
    
    Verify creation:
    
    ```bash
    docker volume ls | grep app_pgdata
    ```
  </Step>

  <Step title="Build and start services">
    Launch the production stack:
    
    ```bash
    docker compose -f docker-compose.prod.yml up -d --build
    ```
    
    Monitor startup:
    
    ```bash
    docker compose -f docker-compose.prod.yml logs -f
    ```
  </Step>

  <Step title="Run database migrations">
    Initialize the database schema:
    
    ```bash
    docker compose -f docker-compose.prod.yml exec backend npm run db:push
    ```
    
    Optionally seed initial data:
    
    ```bash
    docker compose -f docker-compose.prod.yml exec backend npm run db:seed
    ```
  </Step>

  <Step title="Configure reverse proxy">
    Set up Nginx as a reverse proxy (see Nginx Configuration section below).
  </Step>

  <Step title="Enable SSL/TLS">
    Secure your site with Let's Encrypt (see SSL Configuration section below).
  </Step>
</Steps>

## Nginx Reverse Proxy Configuration

In production, use Nginx as a reverse proxy to handle SSL termination and route traffic to Docker services.

### Install Nginx

```bash
sudo apt-get update
sudo apt-get install nginx
```

### Configure Site

Create `/etc/nginx/sites-available/enigmasac.com`:

```nginx /etc/nginx/sites-available/enigmasac.com
upstream frontend {
    server 127.0.0.1:4051;
}

upstream backend {
    server 127.0.0.1:4050;
}

server {
    listen 80;
    server_name enigmasac.com www.enigmasac.com;

    # Redirect HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name enigmasac.com www.enigmasac.com;

    # SSL certificates (configured by Certbot)
    ssl_certificate /etc/letsencrypt/live/enigmasac.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/enigmasac.com/privkey.pem;
    
    # SSL configuration
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 256;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml image/svg+xml;

    # File upload limit
    client_max_body_size 5M;

    # API routes
    location /api/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # Rate limiting
        limit_req zone=api burst=20 nodelay;
    }

    # Frontend routes
    location / {
        proxy_pass http://frontend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Rate limiting zone
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
```

### Enable Site

```bash
sudo ln -s /etc/nginx/sites-available/enigmasac.com /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## SSL Configuration with Let's Encrypt

<Steps>
  <Step title="Install Certbot">
    ```bash
    sudo apt-get install certbot python3-certbot-nginx
    ```
  </Step>

  <Step title="Obtain SSL certificate">
    ```bash
    sudo certbot --nginx -d enigmasac.com -d www.enigmasac.com
    ```
    
    Follow the prompts to configure automatic HTTPS redirection.
  </Step>

  <Step title="Verify auto-renewal">
    Test the renewal process:
    
    ```bash
    sudo certbot renew --dry-run
    ```
    
    Certbot automatically creates a cron job for renewal.
  </Step>
</Steps>

<Note>
  Let's Encrypt certificates are valid for 90 days and auto-renew via cron job.
</Note>

## Monitoring and Logging

### Container Logs

View real-time logs:

```bash
# All services
docker compose -f docker-compose.prod.yml logs -f

# Specific service
docker compose -f docker-compose.prod.yml logs -f backend

# Last 100 lines
docker compose -f docker-compose.prod.yml logs --tail=100 backend
```

### Container Health

Monitor container status:

```bash
# Service status
docker compose -f docker-compose.prod.yml ps

# Detailed container info
docker inspect enigmasac-backend

# Resource usage
docker stats
```

### Database Health

Check PostgreSQL status:

```bash
# Connection test
docker compose -f docker-compose.prod.yml exec db pg_isready -U enigma_user

# Database size
docker compose -f docker-compose.prod.yml exec db psql -U enigma_user -d enigma -c "\l+"

# Active connections
docker compose -f docker-compose.prod.yml exec db psql -U enigma_user -d enigma -c "SELECT count(*) FROM pg_stat_activity;"
```

### Nginx Logs

```bash
# Access logs
sudo tail -f /var/log/nginx/access.log

# Error logs
sudo tail -f /var/log/nginx/error.log
```

## Backup and Maintenance

### Database Backups

#### Manual Backup

Create a database dump:

```bash
docker compose -f docker-compose.prod.yml exec db pg_dump -U enigma_user enigma > backup_$(date +%Y%m%d_%H%M%S).sql
```

#### Automated Backup Script

Create `/opt/enigma/backup.sh`:

```bash backup.sh
#!/bin/bash
BACKUP_DIR="/opt/enigma/backups"
DATE=$(date +%Y%m%d_%H%M%S)
FILE="$BACKUP_DIR/enigma_$DATE.sql"

mkdir -p $BACKUP_DIR

cd /opt/enigma/source
docker compose -f docker-compose.prod.yml exec -T db pg_dump -U enigma_user enigma > $FILE

# Compress backup
gzip $FILE

# Keep only last 30 days
find $BACKUP_DIR -name "enigma_*.sql.gz" -mtime +30 -delete

echo "Backup completed: $FILE.gz"
```

Make executable and schedule:

```bash
chmod +x /opt/enigma/backup.sh

# Add to crontab (daily at 2 AM)
crontab -e
# Add line:
0 2 * * * /opt/enigma/backup.sh
```

#### Restore from Backup

```bash
# Stop backend
docker compose -f docker-compose.prod.yml stop backend

# Restore database
gunzip -c backup_20260305_020000.sql.gz | docker compose -f docker-compose.prod.yml exec -T db psql -U enigma_user enigma

# Restart backend
docker compose -f docker-compose.prod.yml start backend
```

### Volume Backups

Backup the entire PostgreSQL data volume:

```bash
# Create volume backup
docker run --rm \
  -v app_pgdata:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/pgdata_backup_$(date +%Y%m%d).tar.gz -C /data .
```

## Security Checklist

<Warning>
  Complete this checklist before going live:
</Warning>

- [ ] Environment variables use strong, unique passwords (32+ characters)
- [ ] `.env` file has `600` permissions and is not in version control
- [ ] Docker services bind to `127.0.0.1` only, not `0.0.0.0`
- [ ] Nginx reverse proxy is configured with SSL/TLS
- [ ] SSL certificate is valid and auto-renewing
- [ ] CORS origins are explicitly set (no wildcards)
- [ ] Database is not exposed to the internet (no public ports)
- [ ] Firewall allows only ports 80, 443, and SSH
- [ ] SSH is configured with key-based authentication (no password auth)
- [ ] Automated backups are configured and tested
- [ ] Monitoring and alerting are in place
- [ ] Security headers are configured in Nginx
- [ ] Rate limiting is enabled for API endpoints

## Firewall Configuration

Configure UFW (Uncomplicated Firewall):

```bash
# Enable firewall
sudo ufw enable

# Allow SSH
sudo ufw allow 22/tcp

# Allow HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Verify rules
sudo ufw status
```

## Performance Optimization

### Database Connection Pooling

Prisma automatically manages connection pooling. Configure limits in `backend/src/index.js`:

```javascript
const prisma = new PrismaClient({
  datasources: {
    db: {
      url: process.env.DATABASE_URL,
    },
  },
});
```

### Nginx Caching

Add caching for static assets in Nginx config:

```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2)$ {
    proxy_pass http://frontend;
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

## Troubleshooting

### Services won't start

Check for port conflicts:

```bash
sudo netstat -tulpn | grep -E ':(4050|4051)'
```

### Database connection errors

Verify database is healthy:

```bash
docker compose -f docker-compose.prod.yml exec db pg_isready
```

### SSL certificate issues

Renew certificate manually:

```bash
sudo certbot renew
sudo systemctl reload nginx
```

### High memory usage

Increase swap space or upgrade server resources. Monitor with:

```bash
free -h
docker stats
```
