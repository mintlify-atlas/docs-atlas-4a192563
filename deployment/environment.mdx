## Overview

Enigma Developers uses environment variables to configure the application across different environments. All configuration is managed through a `.env` file in the project root.

<Warning>
  Never commit the `.env` file to version control. Use `.env.example` as a template and keep sensitive values secure.
</Warning>

## Environment File Structure

The `.env.example` file provides a template for all required configuration:

```bash .env.example
NODE_ENV=development

BACKEND_PORT=4000

POSTGRES_DB=enigma
POSTGRES_USER=enigma_user
POSTGRES_PASSWORD=change_me_secure_password
DATABASE_URL=postgresql://enigma_user:change_me_secure_password@db:5432/enigma

SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=enigma@enigmasac.com
SMTP_PASS=your_app_password_here
SMTP_FROM="Enigma Developers <enigma@enigmasac.com>"
CONTACT_EMAIL=enigma@enigmasac.com

VITE_API_URL=http://localhost/api
CORS_ORIGIN=http://localhost

NGINX_PORT=80
```

## Variable Reference

### Application Configuration

#### NODE_ENV

**Type**: `string`  
**Default**: `development`  
**Used by**: Backend

Defines the runtime environment for the Node.js backend.

**Values:**
- `development` - Development mode with verbose logging
- `production` - Production mode with optimized performance

```bash
NODE_ENV=production
```

#### BACKEND_PORT

**Type**: `number`  
**Default**: `4000`  
**Used by**: Backend

The port on which the Express server listens inside the container.

```bash
BACKEND_PORT=4000
```

<Note>
  This is the internal container port. The external host port is configured in `docker-compose.yml` (default: 4050).
</Note>

### Database Configuration

#### POSTGRES_DB

**Type**: `string`  
**Default**: `enigma`  
**Used by**: Database, Backend

The name of the PostgreSQL database.

```bash
POSTGRES_DB=enigma
```

#### POSTGRES_USER

**Type**: `string`  
**Default**: `enigma_user`  
**Used by**: Database, Backend

The PostgreSQL database username.

```bash
POSTGRES_USER=enigma_user
```

<Warning>
  In production, use a strong, unique username instead of the default.
</Warning>

#### POSTGRES_PASSWORD

**Type**: `string`  
**Default**: `change_me_secure_password`  
**Used by**: Database, Backend

The PostgreSQL database password.

```bash
POSTGRES_PASSWORD=your_secure_password_here
```

<Warning>
  **Critical Security Requirement**: Always change this to a strong, randomly generated password in production. Use at least 32 characters with mixed case, numbers, and symbols.
</Warning>

#### DATABASE_URL

**Type**: `string` (PostgreSQL connection URI)  
**Used by**: Backend (Prisma)

The complete PostgreSQL connection string used by Prisma ORM.

**Format:**
```
postgresql://[user]:[password]@[host]:[port]/[database]
```

**Development:**
```bash
DATABASE_URL=postgresql://enigma_user:change_me_secure_password@db:5432/enigma
```

**Production:**
```bash
DATABASE_URL=postgresql://prod_user:strong_password@db:5432/enigma
```

<Note>
  The hostname `db` refers to the Docker Compose service name. This allows the backend container to resolve the database container on the internal network.
</Note>

### Email Configuration (SMTP)

The application uses SMTP for sending contact form emails and notifications.

#### SMTP_HOST

**Type**: `string`  
**Default**: `smtp.gmail.com`  
**Used by**: Backend

The SMTP server hostname.

**Common providers:**
```bash
# Gmail
SMTP_HOST=smtp.gmail.com

# SendGrid
SMTP_HOST=smtp.sendgrid.net

# Amazon SES
SMTP_HOST=email-smtp.us-east-1.amazonaws.com
```

#### SMTP_PORT

**Type**: `number`  
**Default**: `587`  
**Used by**: Backend

The SMTP server port.

**Common ports:**
- `587` - STARTTLS (recommended)
- `465` - SSL/TLS
- `25` - Plain (not recommended)

```bash
SMTP_PORT=587
```

#### SMTP_USER

**Type**: `string`  
**Default**: `enigma@enigmasac.com`  
**Used by**: Backend

The SMTP authentication username (usually your email address).

```bash
SMTP_USER=enigma@enigmasac.com
```

#### SMTP_PASS

**Type**: `string`  
**Default**: `your_app_password_here`  
**Used by**: Backend

The SMTP authentication password.

```bash
SMTP_PASS=your_app_password_here
```

<Warning>
  For Gmail, you must use an **App Password**, not your regular account password. Generate one at: https://myaccount.google.com/apppasswords
</Warning>

#### SMTP_FROM

**Type**: `string` (email address with optional display name)  
**Default**: `"Enigma Developers <enigma@enigmasac.com>"`  
**Used by**: Backend

The "From" address for outgoing emails.

```bash
SMTP_FROM="Enigma Developers <enigma@enigmasac.com>"
```

#### CONTACT_EMAIL

**Type**: `string` (email address)  
**Default**: `enigma@enigmasac.com`  
**Used by**: Backend

The email address that receives contact form submissions.

```bash
CONTACT_EMAIL=enigma@enigmasac.com
```

### Frontend Configuration

#### VITE_API_URL

**Type**: `string` (URL)  
**Used by**: Frontend (Vite build)

The base URL for API requests from the frontend.

**Development:**
```bash
VITE_API_URL=http://localhost/api
```

**Production:**
```bash
VITE_API_URL=https://enigmasac.com/api
```

<Note>
  All Vite environment variables must be prefixed with `VITE_` to be exposed to the client-side code.
</Note>

### CORS Configuration

#### CORS_ORIGIN

**Type**: `string` (URL or comma-separated URLs)  
**Used by**: Backend

Allowed origins for Cross-Origin Resource Sharing (CORS).

**Development:**
```bash
CORS_ORIGIN=http://localhost
```

**Production:**
```bash
CORS_ORIGIN=https://enigmasac.com,https://www.enigmasac.com
```

<Warning>
  Never use `*` as the CORS origin in production. Always specify exact domains.
</Warning>

### Nginx Configuration

#### NGINX_PORT

**Type**: `number`  
**Default**: `80`  
**Used by**: Nginx (if using standalone nginx service)

The port on which Nginx listens.

```bash
NGINX_PORT=80
```

## Environment Setup

<Steps>
  <Step title="Copy the example file">
    Create your `.env` file from the template:
    
    ```bash
    cp .env.example .env
    ```
  </Step>

  <Step title="Configure development values">
    For local development, the default values in `.env.example` work out of the box. Only update:
    
    - SMTP credentials (if testing email functionality)
    - POSTGRES_PASSWORD (recommended but optional for local dev)
  </Step>

  <Step title="Configure production values">
    For production deployment, update ALL security-sensitive values:
    
    ```bash
    NODE_ENV=production
    POSTGRES_PASSWORD=[generate strong password]
    DATABASE_URL=postgresql://[user]:[password]@db:5432/enigma
    SMTP_PASS=[your app password]
    VITE_API_URL=https://your-domain.com/api
    CORS_ORIGIN=https://your-domain.com
    ```
  </Step>

  <Step title="Verify configuration">
    Start the services and check logs:
    
    ```bash
    docker-compose up -d
    docker-compose logs backend
    ```
  </Step>
</Steps>

## Development vs Production

### Development Configuration

```bash .env (Development)
NODE_ENV=development
POSTGRES_PASSWORD=simple_password
VITE_API_URL=http://localhost/api
CORS_ORIGIN=http://localhost
SMTP_PASS=test_password  # Optional for dev
```

**Characteristics:**
- Verbose logging enabled
- Simple passwords acceptable
- SMTP can be omitted if not testing emails
- CORS allows localhost

### Production Configuration

```bash .env (Production)
NODE_ENV=production
POSTGRES_PASSWORD=Xy9#mK2$pL8@nQ4!vR7&wT3^zS6*aB1
VITE_API_URL=https://enigmasac.com/api
CORS_ORIGIN=https://enigmasac.com,https://www.enigmasac.com
SMTP_PASS=[gmail app password]
```

**Characteristics:**
- Production mode (optimized, minimal logging)
- Strong, randomly generated passwords
- HTTPS URLs for all endpoints
- Exact domain matching for CORS
- Valid SMTP credentials required

## Security Best Practices

<Warning>
  Follow these security guidelines for production deployments:
</Warning>

### Password Requirements

- **Minimum length**: 32 characters
- **Complexity**: Mix of uppercase, lowercase, numbers, and symbols
- **Uniqueness**: Never reuse passwords across environments
- **Generation**: Use a password manager or `openssl rand -base64 32`

### File Permissions

Ensure the `.env` file has restricted permissions:

```bash
chmod 600 .env
```

### Version Control

Verify `.env` is in `.gitignore`:

```bash .gitignore
.env
.env.local
.env.production
```

### Rotating Secrets

Regularly rotate sensitive credentials:

1. Generate new password
2. Update `.env` file
3. Update `DATABASE_URL` to match
4. Restart services: `docker-compose up -d --force-recreate`

## Troubleshooting

### Backend won't connect to database

**Symptom**: Connection errors in backend logs

**Solution**: Verify `DATABASE_URL` matches `POSTGRES_*` variables:

```bash
# These must match:
POSTGRES_USER=enigma_user
POSTGRES_PASSWORD=mypassword
DATABASE_URL=postgresql://enigma_user:mypassword@db:5432/enigma
```

### Email not sending

**Symptom**: SMTP authentication errors

**Solution**: For Gmail, ensure you're using an App Password, not your account password. Other providers may require specific SMTP settings.

### CORS errors in browser

**Symptom**: "CORS policy" errors in browser console

**Solution**: Ensure `CORS_ORIGIN` matches the exact URL in your browser (including protocol and port):

```bash
# If accessing via http://localhost:4051
CORS_ORIGIN=http://localhost:4051
```

### Frontend can't reach API

**Symptom**: 404 or network errors when calling API

**Solution**: Verify `VITE_API_URL` points to the correct backend URL:

```bash
# Development (through host ports)
VITE_API_URL=http://localhost:4050/api

# Or if using nginx reverse proxy
VITE_API_URL=http://localhost/api
```
