# Internationalization (i18n)

The Enigma Developers frontend implements a custom internationalization system supporting Spanish (default) and English, with URL-based language detection and switching.

## Overview

The i18n system consists of:

- **LanguageContext**: React Context for managing language state and translations
- **useT Hook**: Hook for accessing translations in components
- **LocaleLink**: Language-aware Link component for navigation
- **Translation Files**: JSON files with all UI strings
- **Route Mapping**: Multi-language URL routing

## LanguageContext

The core of the i18n system is in `src/i18n/LanguageContext.jsx`:

```jsx
import { createContext, useContext, useState, useEffect } from "react";
import { useNavigate, useLocation } from "react-router-dom";
import { getLangFromPath, getEquivalentPath } from "./routes.js";
import es from "./locales/es.json";
import en from "./locales/en.json";

const translations = { es, en };

function resolveDotPath(obj, key) {
  return key.split(".").reduce((acc, part) => {
    if (acc && typeof acc === "object" && part in acc) return acc[part];
    return undefined;
  }, obj);
}

const LanguageContext = createContext(null);

export function LanguageProvider({ children }) {
  const location = useLocation();
  const navigate = useNavigate();

  const langFromPath = getLangFromPath(location.pathname);
  const [lang, setLangState] = useState(langFromPath);

  // Update language when URL changes
  useEffect(() => {
    const detected = getLangFromPath(location.pathname);
    if (detected !== lang) {
      setLangState(detected);
    }
  }, [location.pathname]);

  // Update document lang attribute
  useEffect(() => {
    document.documentElement.lang = lang;
  }, [lang]);

  // Redirect to /en if user preference is English
  useEffect(() => {
    if (location.pathname !== "/") return;
    const saved = localStorage.getItem("enigma-lang");
    if (saved === "en") {
      navigate("/en", { replace: true });
    }
  }, []);

  function t(key) {
    const value = resolveDotPath(translations[lang], key);
    return value !== undefined ? value : key;
  }

  function setLang(targetLang) {
    if (targetLang === lang) return;
    localStorage.setItem("enigma-lang", targetLang);
    const currentFull = location.pathname + (location.hash || "");
    const equivalent = getEquivalentPath(currentFull, targetLang);
    setLangState(targetLang);
    document.documentElement.lang = targetLang;
    navigate(equivalent);
  }

  return (
    <LanguageContext.Provider value={{ lang, t, setLang }}>
      {children}
    </LanguageContext.Provider>
  );
}

export function useT() {
  const context = useContext(LanguageContext);
  if (!context) throw new Error("useT must be used within a LanguageProvider");
  return context;
}
```

### Key Features

<AccordionGroup>
  <Accordion title="URL-Based Language Detection">
    The language is detected from the current URL path:
    - `/` or `/servicios` → Spanish ("es")
    - `/en` or `/en/services` → English ("en")
    
    This ensures language persists through page refreshes and deep links.
  </Accordion>
  
  <Accordion title="Dot-Path Translation Keys">
    Translations use dot notation to access nested values:
    
    ```javascript
    function resolveDotPath(obj, key) {
      return key.split(".").reduce((acc, part) => {
        if (acc && typeof acc === "object" && part in acc) return acc[part];
        return undefined;
      }, obj);
    }
    
    // Usage:
    t("hero.headline") // Accesses translations.es.hero.headline
    t("nav.services") // Accesses translations.es.nav.services
    ```
  </Accordion>
  
  <Accordion title="LocalStorage Persistence">
    User language preference is saved to localStorage:
    
    ```javascript
    localStorage.setItem("enigma-lang", targetLang);
    ```
    
    On first visit to `/`, if the saved preference is "en", user is redirected to `/en`.
  </Accordion>
  
  <Accordion title="Document Lang Attribute">
    The HTML `lang` attribute is automatically updated:
    
    ```javascript
    useEffect(() => {
      document.documentElement.lang = lang; // "es" or "en"
    }, [lang]);
    ```
    
    This is important for accessibility and SEO.
  </Accordion>
</AccordionGroup>

## useT Hook

Components access translations using the `useT` hook:

```jsx
import { useT } from "../i18n/LanguageContext.jsx";

export default function MyComponent() {
  const { lang, t, setLang } = useT();
  
  return (
    <div>
      <h1>{t("hero.headline")}</h1>
      <p>{t("hero.description")}</p>
      
      <button onClick={() => setLang(lang === "es" ? "en" : "es")}>
        Switch to {lang === "es" ? "English" : "Español"}
      </button>
    </div>
  );
}
```

**Return Values**:

- `lang`: Current language code ("es" or "en")
- `t(key)`: Function to get translation for a key
- `setLang(targetLang)`: Function to switch language

## LocaleLink Component

For language-aware navigation, use `LocaleLink` instead of React Router's `Link`:

**Location**: `src/i18n/LocaleLink.jsx`

```jsx
import { Link } from "react-router-dom";
import { useT } from "./LanguageContext.jsx";
import { ROUTE_MAP, getEquivalentPath, getLangFromPath } from "./routes.js";

export function localeHref(to, lang) {
  // If 'to' is a route key (e.g., "services"), look it up in ROUTE_MAP
  if (ROUTE_MAP[lang]?.[to] !== undefined) {
    return ROUTE_MAP[lang][to];
  }

  // If 'to' is already a path, convert to target language if needed
  if (typeof to === "string" && to.startsWith("/")) {
    const sourceLang = getLangFromPath(to.split("#")[0]);
    if (sourceLang !== lang) {
      return getEquivalentPath(to, lang);
    }
    return to;
  }

  return to;
}

export default function LocaleLink({ to, children, ...props }) {
  const { lang } = useT();
  const resolvedHref = localeHref(to, lang);

  return (
    <Link to={resolvedHref} {...props}>
      {children}
    </Link>
  );
}
```

### Usage Examples

```jsx
import LocaleLink from "../i18n/LocaleLink.jsx";

// Using route key (automatically resolves to /servicios or /en/services)
<LocaleLink to="services">Services</LocaleLink>

// Using direct path (converts to current language)
<LocaleLink to="/contacto">Contact</LocaleLink>

// With hash fragment
<LocaleLink to="/servicios#desarrollo-web">Web Development</LocaleLink>
```

**Behavior**:

| Current Lang | `to` prop | Resolved href |
|---|---|---|
| es | "/contacto" | "/contacto" |
| en | "/contacto" | "/en/contact" |
| es | "services" | "/servicios" |
| en | "services" | "/en/services" |

## Language Switcher

Implement a language switcher:

```jsx
import { useT } from "../i18n/LanguageContext.jsx";

function LanguageSwitcher() {
  const { lang, setLang } = useT();
  const otherLang = lang === "es" ? "en" : "es";

  return (
    <button onClick={() => setLang(otherLang)}>
      <span className={lang === "es" ? "active" : ""}>ES</span>
      <span>/</span>
      <span className={lang === "en" ? "active" : ""}>EN</span>
    </button>
  );
}
```

**Real Implementation** (from Navbar.jsx:60-77):

```jsx
<button
  onClick={() => setLang(otherLang)}
  className="font-mono text-xs uppercase tracking-[0.1em] text-cream2

>
  <span className={lang === "es" ? "text-red" : ""}>ES</span>
  <span className="text-line">/</span>
  <span className={lang === "en" ? "text-red" : ""}>EN</span>
</button>
```

## Best Practices

<CardGroup cols={2}>
  <Card title="Organized Keys" icon="folder-tree">
    Group translations by section:
    - `nav.*` for navigation
    - `hero.*` for hero section
    - `footer.*` for footer
    - `meta.*` for SEO metadata
  </Card>
  
  <Card title="Consistent Naming" icon="tags">
    Use descriptive, consistent key names:
    - `ctaPrimary`, `ctaSecondary` (not `button1`, `button2`)
    - `headline`, `description` (not `text1`, `text2`)
  </Card>
  
  <Card title="Shared Strings" icon="arrows-repeat">
    Extract commonly used strings:
    ```json
    {
      "common": {
        "loading": "Cargando...",
        "error": "Error",
        "save": "Guardar",
        "cancel": "Cancelar"
      }
    }
    ```
  </Card>
  
  <Card title="Pluralization" icon="list-ol">
    For different plural forms, use separate keys:
    ```json
    {
      "items": {
        "one": "1 item",
        "other": "{{count}} items"
      }
    }
    ```
  </Card>
</CardGroup>

<Warning>
  **Missing Translations**: If a translation key doesn't exist, the `t()` function returns the key itself. This helps identify missing translations during development.
</Warning>
