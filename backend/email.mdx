## Overview

The backend uses **Nodemailer 7** to send email notifications for form submissions. Each controller sends two emails: one to the admin and one confirmation to the user.

## Email Configuration

The email transporter is configured in `src/mail.js`:

```javascript src/mail.js
import nodemailer from "nodemailer";

const port = Number(process.env.SMTP_PORT);

const transporter = nodemailer.createTransport({
  host: process.env.SMTP_HOST,
  port,
  secure: port === 465,
  ...(process.env.SMTP_PASS
    ? { auth: { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS } }
    : { auth: false, tls: { rejectUnauthorized: false } }),
});

export async function sendEmail({ to, subject, html }) {
  try {
    await transporter.sendMail({
      from: process.env.SMTP_FROM,
      to,
      subject,
      html,
    });
  } catch (error) {
    console.error("Error al enviar email:", error);
  }
}
```

### Configuration Features

- **Dynamic Security**: Automatically uses TLS/SSL when port is 465
- **Optional Auth**: Supports both authenticated and unauthenticated SMTP
- **Error Handling**: Logs errors without throwing exceptions
- **HTML Support**: Sends rich HTML emails

<Note>
The `sendEmail` function doesn't throw errors to prevent email failures from blocking form submissions.
</Note>

## Environment Variables

Required SMTP configuration in `.env`:

```bash .env
SMTP_HOST="smtp.gmail.com"
SMTP_PORT=587
SMTP_USER="your-email@gmail.com"
SMTP_PASS="your-app-password"
SMTP_FROM="Enigma Developers <noreply@enigma-developers.com>"
CONTACT_EMAIL="info@enigma-developers.com"
```

### Variable Descriptions

| Variable | Description | Example |
|----------|-------------|--------|
| `SMTP_HOST` | SMTP server hostname | `smtp.gmail.com` |
| `SMTP_PORT` | SMTP port (587 for TLS, 465 for SSL) | `587` |
| `SMTP_USER` | SMTP authentication username | `user@gmail.com` |
| `SMTP_PASS` | SMTP authentication password | `app-password-here` |
| `SMTP_FROM` | Default sender name and email | `Company <no-reply@example.com>` |
| `CONTACT_EMAIL` | Admin email for notifications | `admin@example.com` |

<Warning>
For Gmail, use an **App Password** instead of your regular password. Enable 2FA and generate an app-specific password in your Google Account settings.
</Warning>

## Common SMTP Providers

### Gmail

```bash Gmail Configuration
SMTP_HOST="smtp.gmail.com"
SMTP_PORT=587
SMTP_USER="your-email@gmail.com"
SMTP_PASS="your-16-char-app-password"
```

**Setup Steps:**
1. Enable 2-Factor Authentication in Google Account
2. Go to Security → 2-Step Verification → App passwords

## Error Handling

Email errors are logged but don't block the response:

```javascript src/mail.js
export async function sendEmail({ to, subject, html }) {
  try {
    await transporter.sendMail({
      from: process.env.SMTP_FROM,
      to,
      subject,
      html,
    });
  } catch (error) {
    console.error("Error al enviar email:", error);
    // Does not throw - allows form submission to succeed
  }
}
```

**Rationale:**
- Form submissions should succeed even if email fails
- Data is saved to database regardless of email status
- Admins can check logs for email delivery issues

## Troubleshooting

### Common Issues

**Authentication Failed**
```
Error: Invalid login: 535-5.7.8 Username and Password not accepted
```
→ Use app password, not regular password

**Connection Refused**
```
Error: connect ECONNREFUSED 127.0.0.1:587
```
→ Check `SMTP_HOST` and `SMTP_PORT` values

**TLS/SSL Errors**
```
Error: self signed certificate in certificate chain
```
→ Add `tls: { rejectUnauthorized: false }` for local SMTP servers

**Rate Limiting**
```
Error: 454 4.7.0 Too many login attempts
```
→ Gmail limits: 500 emails/day for free accounts
