## Prerequisites

Before you begin, ensure you have the following installed:

- **Docker** and **Docker Compose** (recommended)
- **Node.js 20+** and **npm** (for local development without Docker)
- **PostgreSQL 16** (only if running without Docker)
- **Git**

## Quick Start with Docker

<Steps>
  <Step title="Clone the repository">
    Clone the Enigma Developers repository to your local machine:

    ```bash
    git clone https://github.com/sdelsolar90/webenigma.git
    cd webenigma
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and configure your settings:

    ```bash
    cp .env.example .env
    ```

    Edit the `.env` file with your actual credentials:

    ```bash .env
    NODE_ENV=development

    BACKEND_PORT=4000

    POSTGRES_DB=enigma
    POSTGRES_USER=enigma_user
    POSTGRES_PASSWORD=change_me_secure_password
    DATABASE_URL=postgresql://enigma_user:change_me_secure_password@db:5432/enigma

    SMTP_HOST=smtp.gmail.com
    SMTP_PORT=587
    SMTP_USER=enigma@enigmasac.com
    SMTP_PASS=your_app_password_here
    SMTP_FROM="Enigma Developers <enigma@enigmasac.com>"
    CONTACT_EMAIL=enigma@enigmasac.com

    VITE_API_URL=http://localhost/api
    CORS_ORIGIN=http://localhost

    NGINX_PORT=80
    ```

    <Note>
      Make sure to update `POSTGRES_PASSWORD`, `SMTP_USER`, `SMTP_PASS`, and `CONTACT_EMAIL` with your actual values.
    </Note>
  </Step>

  <Step title="Start the application">
    Launch all services using Docker Compose:

    ```bash
    docker compose up -d
    ```

    This command will:
    - Build the frontend, backend, and nginx containers
    - Start a PostgreSQL 16 database
    - Set up the reverse proxy configuration
    - Expose the application on port 80
  </Step>

  <Step title="Run database migrations">
    Initialize the database schema with Prisma:

    ```bash
    docker compose exec backend npx prisma migrate dev
    ```

    This creates the necessary tables for contacts, leads, complaints, posts, and categories.
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    ```
    http://localhost
    ```

    The Enigma Developers website should now be running!
  </Step>
</Steps>

## Alternative: Local Development Setup

If you prefer to run the services without Docker, follow these steps:

<CodeGroup>

```bash Backend Setup
# Terminal 1 - Backend
cd backend
npm install
npx prisma migrate dev
npm run dev
```

```bash Frontend Setup
# Terminal 2 - Frontend
cd frontend
npm install
npm run dev
```

</CodeGroup>

<Warning>
  For local development, you'll need to:
  1. Have PostgreSQL 16 running locally
  2. Update `DATABASE_URL` in your `.env` file to point to your local PostgreSQL instance
  3. The frontend will run on `http://localhost:5173` and backend on `http://localhost:4000`
</Warning>

### Local Development Configuration

When running locally without Docker, update your `.env` file:

```bash .env
DATABASE_URL=postgresql://enigma_user:your_password@localhost:5432/enigma
VITE_API_URL=http://localhost:4000/api
CORS_ORIGIN=http://localhost:5173
```

## Verifying the Installation

Once your application is running, verify that all services are working:

<Steps>
  <Step title="Check backend health">
    ```bash
    curl http://localhost/api/health
    ```

    Expected response:
    ```json
    {"status":"ok"}
    ```
  </Step>

  <Step title="Check running containers">
    If using Docker, verify all containers are running:

    ```bash
    docker compose ps
    ```

    You should see three containers running:
    - `enigmasac-frontend` (port 4051:80)
    - `enigmasac-backend` (port 4050:4000)
    - `enigmasac-db` (PostgreSQL 16)
  </Step>

  <Step title="Access Prisma Studio (optional)">
    To view and manage your database:

    ```bash
    # With Docker
    docker compose exec backend npx prisma studio

    # Local development
    cd backend && npx prisma studio
    ```

    Prisma Studio will open at `http://localhost:5555`
  </Step>
</Steps>

## Troubleshooting

### Database Connection Issues

If you see database connection errors:

```bash
# Check if PostgreSQL is running
docker compose ps db

# View database logs
docker compose logs db

# Restart the database
docker compose restart db
```

### Port Already in Use

If port 80 is already in use, you can change the nginx port in `.env`:

```bash
NGINX_PORT=8080
```

Then restart the services:

```bash
docker compose down
docker compose up -d
```

### SMTP Configuration

For Gmail SMTP, you'll need to:
1. Enable 2-factor authentication on your Google account
2. Generate an App Password at https://myaccount.google.com/apppasswords
3. Use the App Password in your `SMTP_PASS` variable
